**No Layout System**
- **No `layout.html`** or shared layout fragments
- **No Thymeleaf fragments** (`th:fragment`, `th:replace`)

```

├── Full <!DOCTYPE html>

├── Complete <body>
│   ├── Header (duplicated across pages)
│   ├── Main content
│   └── Footer (duplicated across pages)
└── Inline <style> and <script> tags
```

## 4. API Calling Patterns

### Overview

All API calls are **client-side JavaScript** using `fetch()` API. No server-side API calls in templates.

### Configuration

File: `C:\Users\wtme4\OneDrive\바탕 화면\aitool\teamprojectv1\frontend-service\src\main\resources\static\js\config.js`

```javascript
// API Configuration
const API_CONFIG = {
    baseURL: '',  // Relative paths (works in all environments)

    endpoints: {
        // Auth
        login: '/api/auth/login',
        register: '/api/auth/register',
        logout: '/api/auth/logout',
        findUserId: '/api/auth/find-userid',
        findPassword: '/api/auth/find-password',
        emailSend: '/api/auth/email/send-code',
        emailVerify: '/api/auth/email/verify-code',

        // User
        userProfile: '/api/users/profile',

        // Admin
        adminUsers: '/api/admin/users',
        adminNotices: '/api/admin/notices',
        adminInquiries: '/api/admin/inquiries',
        adminProducts: '/api/admin/products',

        // Products
        products: '/api/products',

        // Orders
        orders: '/api/orders'
    }
};

// Helper: Get full API URL with params
function getApiUrl(endpoint, params = {}) {
    let url = API_CONFIG.baseURL + endpoint;

    const queryString = Object.keys(params)
        .filter(key => params[key] !== null && params[key] !== undefined)
        .map(key => `${encodeURIComponent(key)}=${encodeURIComponent(params[key])}`)
        .join('&');

    if (queryString) {
        url += (url.includes('?') ? '&' : '?') + queryString;
    }

    return url;
}

// Helper: Get Authorization header
function getAuthHeader() {
    const token = localStorage.getItem('accessToken');
    return token ? `Bearer ${token}` : null;
}

// Helper: Authenticated fetch
function authenticatedFetch(url, options = {}) {
    const authHeader = getAuthHeader();

    if (!authHeader) {
        window.location.href = '/login';
        return Promise.reject(new Error('인증이 필요합니다.'));
    }

    const headers = {
        'Content-Type': 'application/json',
        'Authorization': authHeader,
        ...(options.headers || {})
    };

    return fetch(url, {
        ...options,
        headers: headers
    });
}
```

```javascript
function fetchMenuList(page = 1) {
    const params = new URLSearchParams();
    params.set("page", page);
    params.set("limit", PAGE_SIZE);

    // Add category filters
    selectedCategories.forEach(cat => {
        params.append("category", cat);

            $('#menuCount').text(data.totalElements ?? 0);
```

### ❌ Bad Patterns / Anti-Patterns

**1. Code Duplication (Header/Footer in Every Template)**

❌ **Problem:** Header and footer HTML duplicated across 35+ templates

**Impact:**
- Maintenance nightmare
- Inconsistency risk
- Update requires editing 35+ files

**Better Approach:** Use Thymeleaf fragments
```html
<!-- layout/header.html -->
<header th:fragment="header">
    <!-- Header content -->
</header>

<!-- In each page -->
<div th:replace="~{layout/header :: header}"></div>
```

**2. Inline Styles in Templates**

File: `C:\Users\wtme4\OneDrive\바탕 화면\aitool\teamprojectv1\frontend-service\src\main\resources\templates\order\cart.html` (lines 27-663)

❌ **Problem:** 600+ lines of CSS in `<style>` tag

**Impact:**
- Not reusable
- Harder to maintain
- No caching benefits

**Better Approach:** Extract to separate CSS file
```html
<link rel="stylesheet" th:href="@{/css/cart.css}">
```

**3. Global Function Pollution**

File: `C:\Users\wtme4\OneDrive\바탕 화면\aitool\teamprojectv1\frontend-service\src\main\resources\templates\order\cart.html` (lines 1068-1103)

❌ **Problem:** Functions attached to `window` object
```javascript
window.updateQty = async function(cartItemId, newQuantity) { ... }
window.deleteItemFromServer = async function(cartItemId) { ... }
```

**Impact:**
- Global namespace pollution
- Naming conflicts
- Hard to track dependencies

**Better Approach:** Module pattern
```javascript
const CartModule = (function() {
    async function updateQty(cartItemId, newQuantity) { ... }
    async function deleteItem(cartItemId) { ... }

    return { updateQty, deleteItem };
})();
```

**4. Hardcoded API URLs**

File: `C:\Users\wtme4\OneDrive\바탕 화면\aitool\teamprojectv1\frontend-service\src\main\resources\templates\menu\drink.html` (line 800)

❌ **Problem:** API URL hardcoded in template

**5. No Error Boundaries**

❌ **Problem:** Errors crash entire page

```javascript

    });
```

```javascript
        errorMessage = '비밀번호를 입력하세요';
        showCustomModal('알림', errorMessage);

    }
```

## Summary & Best Practices

### ✅ Do's

1. **Use Design Tokens** from `global-ui-kit.css` for consistent spacing, colors, typography
2. **Include Config.js** for centralized API endpoint management
3. **Always Handle Errors** in fetch() calls with `.catch()`
4. **Use SweetAlert2** for confirmation dialogs (better UX than custom modals)
5. **Validate Client-Side** before sending to API (email format, required fields)
6. **Check Login Status** on every protected page load
7. **Use Toast Notifications** for non-blocking user feedback
8. **Add Loading States** for async operations (disable buttons, show spinners)
9. **Implement Fallback Images** with `onerror="this.src='/fallback.png'"`
10. **Use AOS** for scroll animations (already included)

### ❌ Don'ts

1. **Don't Duplicate Header/Footer** - Consider Thymeleaf fragments
2. **Don't Put 600+ Lines of CSS** in `<style>` tags - Extract to separate files
3. **Don't Pollute Global Scope** with `window.functionName` - Use modules
4. **Don't Hardcode API URLs** - Use config.js
5. **Don't Skip Error Handling** - Always add `.catch()` to promises
6. **Don't Use `alert()`** - Use custom modals or SweetAlert2
7. **Don't Ignore Mobile** - Test on 768px breakpoint
8. **Don't Mix Naming Conventions** - Pick BEM or underscore_case, stick with it
9. **Don't Forget ARIA Labels** for accessibility
10. **Don't Use Deprecated jQuery Methods** (`.live()`, `.bind()`) - Use `.on()`

### Key Takeaways

- **SSR-First Architecture:** Templates rendered server-side, enhanced with JavaScript
- **No Modern Build Tools:** Direct file serving, no bundling
- **jQuery + Vanilla JS:** Mix of jQuery for DOM manipulation, vanilla JS for modern APIs
- **Design System in Place:** CSS custom properties in `global-ui-kit.css`
- **Brand Consistency:** `v2_theme.css` overrides legacy yellow with TORI COFFEE orange (#D95B3B)
- **Code Duplication Issue:** Header/footer duplicated across 35 templates
- **Client-Side State:** localStorage for auth, no complex state management
- **Fetch-Based APIs:** All API calls use `fetch()`, no axios or jQuery.ajax()
